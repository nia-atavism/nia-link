# Security Policy

## Reporting a Vulnerability

We take the security of Nia-Link seriously. If you believe you have found a security vulnerability, please report it to us by emailing linianrou1020@gmail.com.

Please include the following information in your report:
- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

We prefer all communications to be in English or Traditional Chinese.

## Response Process

After we receive your report, we will:
1. Acknowledge receipt of your report within 48 hours.
2. Investigate the issue and confirm its validity.
3. Work on a fix and release a security update.
4. Notify you once the issue has been resolved.

Thank you for helping us keep Nia-Link secure! 🎧🛡️
